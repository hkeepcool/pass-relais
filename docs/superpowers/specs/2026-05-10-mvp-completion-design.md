## Feature 1 — PatientDetailPage (`/patients/:id`)

### Layout

Single scrollable screen, no tabs:

```
[← Back]  [Patient full name]
──────────────────────────────────────
Section: Sommeil
  [Reposé]  [Agité]  [Insomnie]        ← QuickTapButton row, tone: ok/warn/alert

Section: Appétit
  [Normal]  [Faible]  [Refus]          ← tone: ok/warn/alert

Section: Douleur (pain 1–5)
  [1 ☆]  [2 ☆☆]  [3 ☆☆☆]  [4]  [5]   ← tone: ok/ok/warn/warn/alert
  glyph chars: ①②③④⑤ or face glyphs

Section: Humeur
  [Stable]  [Confus]  [Anxieux]        ← tone: ok/warn/warn

──────────────────────────────────────
[VoiceButton]  transcription → note_text

──────────────────────────────────────
[Enregistrer]  Button variant=accent, fullWidth, size=lg
               disabled until at least one field is selected
               loading=true while saving
```

### Design system usage

- Section headers: plain `text-ink-mute text-xs font-semibold uppercase tracking-widest` (Tailwind token, no custom component needed)
- Each option row: `<div role="radiogroup">` wrapping `QuickTapButton` components with appropriate `tone` prop
- `VoiceButton` in `idle → recording → done` cycle fed by `useTranscription`
- Submit: `<Button variant="accent" fullWidth size="lg" loading={isSaving}>`
- Back navigation: `<Button variant="ghost" size="sm">` with chevron leading icon

### Status color derivation (auto, no caregiver input)

```
pain === 5                          → 'red'
pain === 4 || mood === 'confused'
           || mood === 'anxious'    → 'orange'
else                                → 'green'
```

## Feature 2 — `useTranscription` Hook

### File

`src/shared/hooks/useTranscription.ts`

### Interface (stable — never changes regardless of adapter)

```ts
interface TranscriptionResult {
  state: 'idle' | 'recording' | 'done' | 'unsupported'
  transcript: string
  duration: number          // elapsed seconds
  start: () => void
  stop: () => void
  reset: () => void
}
```

### Adapter pattern

```ts
interface TranscriptionAdapter {
  start(onInterim: (t: string) => void, onFinal: (t: string) => void): void
  stop(): void
  isSupported(): boolean
}
```

Two implementations (both in `src/shared/hooks/transcription/`):

**`WebSpeechAdapter.ts`** (ships now)
- Uses `window.SpeechRecognition || window.webkitSpeechRecognition`
- `continuous: true`, `interimResults: true`, `lang: 'fr-FR'`
- `onresult` → fires `onInterim` on interim, `onFinal` on final transcript
- `onspeechend` → calls `stop()`

**`WhisperAdapter.ts`** (stub — add later)
- Uses `MediaRecorder` to capture audio blob
- On `stop()` → POST blob to Supabase Edge Function `transcribe-audio`
- Edge Function calls OpenAI Whisper API, returns `{ text }`
- `isSupported()` always returns true (connectivity checked at call site)

**Swap:** Change one import in `useTranscription.ts` to switch adapters. No consumer changes.

### Graceful degradation

If `adapter.isSupported()` returns false, `state` is permanently `'unsupported'`. `PatientDetailPage` hides the `VoiceButton` section entirely in this case.

## Feature 3 — FeedPage (`/patients/:id/feed`)

### Layout

```
[← Back]  [Patient full name]
──────────────────────────────────────
[AI Summary card]
  Badge status=tone  "Résumé des dernières 24h"
  text: generated by summarize()

──────────────────────────────────────
Filter chips (horizontal scroll):
  [Tout]  [Incidents]  [Notes]

──────────────────────────────────────
[FeedEntry] × N   — chronological, newest first
  isLast={index === filtered.length - 1}
```

### Design system usage

- Summary card: `bg-surface border border-line rounded-xl px-4 py-3` + `Badge` for tone
- Filter chips: `<Button variant="secondary" size="sm">` — active state via `variant="accent"`
- Feed items: `<FeedEntry>` with all props populated from `Observation` data
- Empty state: centered `text-ink-mute` paragraph, no additional component

### `FeedEntry` prop mapping from `Observation`

```ts
{
  timestamp:      format(obs.recorded_at, 'HH:mm'),
  authorName:     obs.caregiver_id === currentUserId ? 'Vous' : obs.caregiver_id.slice(0, 8),
  authorInitials: initials(caregiver.full_name),
  text:           buildText(obs),              // see below
  status:         colorToTone(obs.status_color),
  tag:            obs.status_color === 'red'
                    ? { label: 'Urgent', status: 'alert' }
                    : obs.status_color === 'orange'
                    ? { label: 'Vigilance', status: 'warn' }
                    : undefined,
  metrics:        buildMetrics(obs),           // see below
}
```

`buildText(obs)` — concatenates non-null fields into one sentence. `note_text` appended if present.  
`buildMetrics(obs)` — returns `FeedMetric[]` for sleep/appetite/pain/mood as label+value pairs, only for non-null fields.

### Filters

```
'all'       → all observations
'incidents' → obs.status_color === 'orange' || 'red'
'notes'     → obs.note_text !== null || obs.note_audio_url !== null
```

Filter state is local (`useState`) — not persisted, resets on page leave.

### Rule-based summary — `src/shared/utils/summarize.ts`

```ts
export function summarizeObservations(obs: Observation[]): { text: string; tone: StatusTone }
```

Reads observations from last 24h:
- Finds most-recent value per field (sleep, appetite, pain, mood)
- Maps to French sentence fragments
- Concatenates into 1–3 sentences
- Returns `tone` = worst status_color across window (mapped to StatusTone)

**Upgrade path:** Replace the function body with a `fetch` to Supabase Edge Function `summarize-observations` calling Claude Haiku. The signature and return type stay identical — zero consumer changes.
